# Long Horizon 设计调查：从人类远见者到 Agent 长时记忆

**日期**: 2026-05-13
**状态**: 调查中
**目标**: 为 Axis 的 long horizon 记忆系统建立设计依据

---

## 核心问题

Axis 的 long horizon 不是"存更多历史"，而是**让 Agent 拥有远见者的决策质量**。

---

## 一、人类远见者的 Long Horizon 结构

### 三层载体

| 维度 | 人类远见者 | 长视野 Agent |
|------|-----------|-------------|
| **Patterns 模式** | 潜意识底层记忆、规律萃取 | 向量记忆 + 世界模型，自动萃取任务模式、行为模式、失败模式，是 Agent 长时决策底层 |
| **Principles 原则** | 显性信条、决策准则 | System Prompt / 约束规则 / 第一性规约，做规划校验、行为边界、价值对齐 |
| **Narrative 叙事** | 人生故事、自我叙事 | 任务日志 + 过程复盘文本，用来做链式复盘、任务回溯、上下文连贯性缝合 |

### 远见者 vs 短视者的区别

| 维度 | 远见者 | 短视者 |
|------|--------|--------|
| 模式层 | 从失败中提取可复用模式 | 记住具体事件但不抽象 |
| 原则层 | 少量坚定原则指导一切决策 | 无原则或原则互相矛盾 |
| 叙事层 | 连贯的自我叙事，知道"我从哪来到哪去" | 碎片化，每天都是新的一天 |
| 遗忘 | 主动遗忘细节，保留模式 | 要么全忘，要么被细节淹没 |
| 决策 | 原则已内化为直觉，不需要显式回忆 | 每次决策都从零开始 |

### 主要矛盾

**造成远见/短视区别的主要矛盾**：是否能将经验**蒸馏为可复用的抽象**（模式+原则），并**遗忘原始细节**。

短视者的失败不是"记忆力差"，而是**蒸馏能力差**——他们记住了太多无用细节，却没有提取出模式。

## 二、可迁移到 Axis 的设计启示

### 直接迁移

| 人类机制 | Axis 设计 | 实现方式 |
|----------|-----------|----------|
| 模式蒸馏 | Auto-Dream 回放 → 提取 patterns | `.axis/memory/learned/patterns.md` |
| 原则内化 | 原则写入 System Prompt，决策时不检索 | CLAUDE.md / Contract / Skills Layer 1 |
| 时间衰减分辨率 | 近时域高保真，远时域低分辨率 | events.jsonl 保留原始；dream 产出概要 |
| 里程碑锚定 | 关键事件高精度保留 | `.axis/memory/milestones/` 特例存储 |
| 主动遗忘 | 只留模式/原则，丢弃事件细节 | dream 后原始记录可归档/压缩 |
| 按需检索 | 仅未知场景/高风险/模式失效时检索 | Agent 调用 `recall_memory` tool，非自动注入 |

### 无关因素（不迁移）

| 人类机制 | 为什么不迁移 |
|----------|-------------|
| 情绪记忆（恐惧/兴奋强化记忆） | Agent 无情绪，用"失败严重性"替代 |
| 社会叙事（"我是谁"的身份认同） | Agent 无自我意识需求，叙事层仅用于任务连贯性 |
| 睡眠整合（REM 阶段记忆重组） | 用 auto-dream 的离线回放替代，但不需要模拟睡眠周期 |
| 直觉/体感（"感觉不对"） | 用模式匹配的置信度分数替代 |

## 三、Axis Long Horizon 架构设计

### 存储层（半结构化：Unix files + markdown + Agent 驱动更新）

```
.axis/memory/
├── patterns/              # 模式层（Agent 决策底层）
│   ├── success.md         # 成功模式：什么条件下什么策略有效
│   ├── failure.md         # 失败模式：什么条件下什么会失败
│   └── behavioral.md      # 行为模式：Agent 自身的行为规律
├── principles/            # 原则层（已内化，注入 System Prompt）
│   └── derived.md         # 从经验中蒸馏的原则（区别于人工写的 CLAUDE.md）
├── narrative/             # 叙事层（任务连贯性）
│   ├── timeline.md        # 时间线概要（远时域低分辨率）
│   └── milestones/        # 里程碑锚定（高精度特例）
├── dreams/                # Dream 回放记录
│   └── 2026-05-13-001.md
└── index.md               # 记忆索引
```

### 运行时行为

```
日常决策（99% 场景）：
├── System Prompt 注入 principles/derived.md（已内化，零检索成本）
├── Skills Layer 1 注入 patterns/ 摘要（~200 tokens）
└── 不检索原始历史，不注入完整记忆

按需检索（1% 场景：未知/高风险/模式失效）：
├── Agent 主动调用 recall_memory tool
├── 检索 narrative/milestones/ 中的关键片段
└── 检索 immunity memory 中的相关失败记录

离线蒸馏（Dream 机制）：
├── 回放最近 N 个任务的 events.jsonl
├── 提取新模式 → 更新 patterns/
├── 蒸馏新原则 → 更新 principles/derived.md
├── 压缩叙事 → 更新 narrative/timeline.md
└── 标记里程碑 → 写入 narrative/milestones/
```

### 遗忘策略

| 时域 | 保留内容 | 遗忘内容 |
|------|----------|----------|
| <24h | 完整 events.jsonl | 无 |
| 1-7d | 任务级概要 + 关键 tool 输出 | 具体 stdout、中间状态 |
| 7-30d | 模式 + 原则 + 里程碑 | 任务级概要 |
| >30d | 仅 patterns/ + principles/ + milestones/ | 一切具体事件 |

## 四、失败在 Long Horizon 中的角色（来源：专题报告）

### 核心映射：压抑→反刍→免疫

| 人类机制 | 含义 | Axis 映射 |
|----------|------|-----------|
| **压抑** | 主动选择不存储无价值失败 | 遗忘策略：低价值失败不进入 patterns/，只保留摘要 |
| **反刍** | 反复分析失败原因（有害的循环） | **反面教材**：Agent 不应无限重试同一策略。MaxRetryLimit=3 + 强制回退 |
| **免疫转化** | 从失败中提取防御能力 | immunity memory：失败特征→签名→快速匹配→自动应对 |

### 关键设计决策

1. **失败是资产，不是噪声**：每次失败都应被评估"是否可蒸馏为模式"，而非简单丢弃
2. **反刍有毒**：系统不应对同一失败反复分析。Dream 机制处理一次后，结论写入 patterns/，原始记录归档
3. **免疫分层**：
   - L1 基础免疫：已知失败类型的快速匹配（immunity memory 现有实现）
   - L2 高级免疫：需要模式推理的失败（patterns/ 中的规律）
   - L3 自适应免疫：全新失败类型的处理（触发 dream 回放 + 人类介入）
4. **失败传播阻断**：早期小错误会滚雪球。verify_bash + 退出码验证 = 传播阻断点
5. **垃圾记忆过滤标准**：
   - 冗余（已有相似模式）→ 丢弃
   - 过时（环境已变）→ 标记 deprecated
   - 错误（幻觉产物）→ 立即删除
   - 低价值（一次性问题）→ 不进入长期记忆

### 对 Dream 机制的具体要求

Dream 回放时必须执行：
1. **聚类**：将相似失败归为一组
2. **蒸馏**：从每组中提取可复用模式（不是保留每个细节）
3. **验证**：新模式是否与已有模式矛盾？矛盾则触发 Sublation
4. **遗忘**：蒸馏完成后，原始失败细节可归档压缩
