# 里程碑 1 技术准入标准检查清单（基础版）

## 里程碑目标
聚焦 Agent 调度核心能力验证，实现基础任务调度、简单任务编排、上下文管理的最小闭环。

**注意**：本里程碑采用基础版方案，聚焦 Agent 调度核心能力。完整架构设计见 `feature/full-architecture` 分支。

**重要说明**：
- **终态**：Agent 原生调度系统
- **CLI 定位**：CLI 只是调度系统的一个客户端，不是核心
- **里程碑 1 范围**：基础 Agent 调度（FIFO + 简单依赖管理）

## 技术准入标准（基础版）

### 1. 基础任务调度（核心能力 - 必须保留）

#### 1.1 任务队列验证
- [ ] FIFO 任务队列实现
- [ ] 任务提交/消费正常
- [ ] 任务队列容量 1000 无阻塞
- [ ] 任务状态跟踪（pending/running/completed/failed）

#### 1.2 调度策略验证
- [ ] FIFO 调度策略验证通过
- [ ] 任务串行执行验证通过

### 2. 简单任务编排（核心能力 - 必须保留）

#### 2.2 任务编排执行

- [ ] 编排结果返回验证通过

### 3. 上下文管理（核心能力 - 必须保留）

## 验收流程（基础版）

### 阶段 1：自检
- 完成所有检查项自检
- 生成自检报告

### 阶段 2：自动化测试
- 运行测试套件
- 生成测试报告

### 阶段 3：手动验收
- 基础功能验证
- 里程碑 1 正式准入

## 失败处理

### 检查项失败处理
- 单项失败：修复后重新验证
- 阻塞性失败：暂停准入，根因分析

### 降级准入
- 非阻塞性检查项失败可申请降级准入
- 降级准入需明确补救计划和时间表
- 降级准入需技术委员会批准
