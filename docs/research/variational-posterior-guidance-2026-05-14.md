# Variational Posterior Guidance：对 Axis 上下文压缩的启示

> 基于 arXiv:2605.11019 — Chen et al., 2026-05-10

## 1. 核心机制（≤10行）

LLM 推理中 Chain-of-Thought 存在"过度思考"问题——生成冗长推理链却不提升正确率。
VPG-EA 将高效推理形式化为**变分推断问题**：
- **后验分布**：以参考答案为条件，理论证明其期望效用高于先验分布（即无条件生成）
- **效率感知 ELBO**：引入效率项的证据下界作为优化目标
- **参数共享双流架构**：同一模型同时实例化后验分布和先验策略
- **交叉视角评估**：过滤伪高效路径（短但错的推理链）
- **变分蒸馏**：单向将后验的高效模式迁移到先验策略
- 在 1.5B/7B 模型上，综合效率指标 ε³ 分别提升 8.73% / 12.37%

## 3. 对 Axis 的启示

- Axis 的三层压缩（micro/auto/manual）是启发式的，没有理论保证
- VPG-EA 证明：后验分布引导的压缩比先验策略更优——知道答案方向时，压缩路径质量显著提升
- "压缩不是丢信息，是找等价的短路径"——这个洞见对 Axis 的 SummarizationCompaction 有指导意义
- 可借鉴：**质量感知的压缩**（保留对任务有用的信息，不是简单截断）
  - 压缩时引入任务目标作为"伪后验"信号，评估哪些上下文片段对当前任务有贡献
  - 交叉视角验证：压缩后让 LLM 判断是否丢失关键信息
- 不能借鉴：双流架构 + RL 训练（Axis 不训练模型，只调用 API）

## 4. 可行动建议

1. **ContextBundle 压缩增加任务感知评分**：压缩前用 task intent 对每个 context chunk 打相关性分数，优先保留高分段
2. **压缩后验证门**：压缩结果经 LLM 快速检查"是否保留了完成任务所需的关键信息"，类似 VPG-EA 的交叉视角过滤
3. **记录压缩质量指标**：在 ExecutionContextSummary 中追踪压缩比 vs 任务成功率，积累数据验证压缩策略有效性
4. **不做**：不引入 RL 训练、不修改模型权重、不增加双流推理开销
