# Agent 原生调度系统白皮书 v1.1

本项目旨在为 AI Agent 提供统一的任务调度能力。

## 一、问题

现有 Agent 工具缺乏统一的调度系统，导致：
- Agent 无法调度其他 Agent
- Agent 无法调度人类任务
- 多 Agent 协作缺乏依赖管理

## 二、解决方案

Agent 原生调度系统是一个面向 AI Agent 的统一调度平台，提供以下核心能力：

### 2.1 任务调度
- FIFO 任务队列
- 任务依赖管理
- 任务状态跟踪

### 2.2 契约验证
- 输入 Schema 验证
- 输出 Schema 验证

### 2.3 状态管理
- 任务状态持久化
- 状态查询

### 2.4 人类任务队列
- 人类任务提交
- 任务执行跟踪

## 三、架构设计

### 3.1 核心模块
- **调度器**：FIFO 任务调度 + 简单依赖管理
- **分发器**：任务分发到执行器
- **契约执行器**：输入输出验证
- **Human 执行器**：人类任务队列
- **状态存储**：内存状态管理

### 3.2 客户端
- CLI 客户端（便于人类交互）
- API 客户端（便于 Agent 调用，里程碑2）
- SDK 客户端（便于集成，里程碑2）

**注意**：CLI 只是客户端之一，不是核心。

## 四、里程碑1目标

聚焦核心能力验证，实现最小闭环：

### 4.1 包含
- ✅ FIFO 任务调度
- ✅ 简单依赖管理（依赖完成才执行）
- ✅ 输入输出 Schema 验证
- ✅ 基础状态存储
- ✅ 基础 CLI

### 4.2 不包含（延后到里程碑2+）
- ❌ DAG 并行调度
- ❌ 契约准入规则
- ❌ SLA 约定（超时、重试、熔断）
- ❌ 异常码体系
- ❌ 工具调用层
- ❌ 全局事件总线

## 五、演化路径

### 里程碑1：基础调度
- FIFO 任务调度
- 简单依赖管理
- 输入输出验证

### 里程碑2：并行调度
- DAG 并行调度
- 契约准入规则
- SLA 约定

### 里程碑3：生态成熟
- 工具调用层
- 完整异常处理
- 多客户端支持

## 六、总结

Agent 原生调度系统，是面向 AI Agent 的统一调度平台，核心是提供任务调度、依赖管理、输入验证能力，支持 Agent 调度其他 Agent 和人类任务。

CLI 只是调度系统的一个客户端，不是核心。
