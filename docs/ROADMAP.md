# 项目演化路线图

## 三阶段演化

```
里程碑1 → 里程碑2 → 里程碑3
基础调度   并行调度   生态成熟
```

## 里程碑1：基础调度（当前）

**核心能力**
- FIFO 任务调度
- 简单依赖管理
- 输入输出验证
- 基础状态存储
- 基础 CLI

**验证标准**
- 任务提交 → 调度 → 执行 → 结果返回 闭环
- 依赖任务完成后才执行后续任务
- 输入输出 Schema 验证通过

## 里程碑2：并行调度

**新增能力**
- DAG 并行调度
- 契约准入规则（本地/远程校验）
- SLA 约定（超时、重试、熔断）
- 异常码体系

**性能提升**
- 无依赖任务 100% 并行执行
- 总耗时从「各环节时长之和」压缩为「关键路径时长」

## 里程碑3：生态成熟

**新增能力**
- 工具调用层
- 完整异常处理
- 多客户端支持（API、SDK）
- 契约版本管理
- 契约库

**生态价值**
- 第三方集成
- 契约复用
- 完整的 Agent 调度生态

## 演化原则

- 每个里程碑独立可验证
- 后续里程碑不修改核心调度语义
- 保持向后兼容
- 奥卡姆剃刀：最小可行，渐进增强
