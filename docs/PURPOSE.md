# Axis Documentation Purpose

> 展开自 CLAUDE.md §11（演化原则）

## 为什么存在

为 Axis 项目的所有参与者（人类 + Agent）提供**可查询、可验证、自维护**的知识基础设施。

文档不是装饰。文档是 Agent 的工作记忆外化——编译一次，持续维护，而非每次查询重新推导。

## 服务对象

1. **Agent**（首要）：新会话启动时，通过 CLAUDE.md → docs/README.md → 具体文档 的路径获取上下文
2. **人类开发者**：理解设计决策、当前状态、下一步方向
3. **未来的 Axis 自身**：自我演化时的参照基线

## 关键问题（每个目录回答一个）

| 目录 | 回答的问题 |
|------|-----------|
| `architecture/` | 为什么这样设计？边界在哪？ |
| `specs/` | 下一步具体做什么？契约是什么？ |
| `research/` | 外部世界有什么值得吸收的？ |
| `status/` | 当前到什么程度？什么已验证？ |
| `guides/` | 如何开始工作？方法论是什么？ |

## 演化方向

```
被动文档 → 主动知识库 → 自维护系统
     ↑ 当前位置        ↑ 目标
```

- **当前**：Agent 读文档，人类写文档
- **目标**：Agent 读文档，Agent 也维护文档（在人类审核下）
- **终态**：文档系统自我检测不一致、自我修复、自我演化

## 运营原则

1. **编译优于检索**：知识整合进文档，不散落在聊天历史中
2. **一个事实一个位置**：避免重复声明，用引用代替复制
3. **可验证优于可读**：frontmatter 元数据支持程序化检查
4. **过时即删除**：deprecated/ 是垃圾桶，不是博物馆
