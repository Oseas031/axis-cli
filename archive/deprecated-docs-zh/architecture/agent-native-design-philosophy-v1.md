# Agent 原生设计哲学

## 核心命题

**More Context, More Action, Zero Control, Controllable Evolution**

Axis 相信 Agent 需要的不是更密集的外部控制，而是：

- **More Context**：获得足够理解任务、依赖、历史、系统状态和失败原因的上下文。系统提供高效查询基础设施，不主动推送冗余信息。
- **More Action**：拥有执行、组合、验证、修正和生成后续任务的行动能力，并获得与能力匹配的权限和决策权。
- **Zero Control**：系统提供契约、基础设施和可观测性，但不替 Agent 规定唯一行动路径，不干预 Agent 的自主决策与自我演化。
- **Controllable Evolution**：Agent 的自举、自生成过程必须处于可控范围，通过思想注入校验、自我修改校验、权限约束和审计机制确保演化方向正确、风险可控。

## 权限哲学

**Competence earns autonomy, autonomy matches responsibility, evolution is controllable**

- **Competence**：能力不是抽象智能，而是在真实任务中展现出的可靠执行能力，以任务完成率、失败复盘质量、风险控制能力等量化指标动态评估。
- **Earns**：自主权通过稳定、可验证、可审计的表现逐步赢得，权限范围与能力水平精准匹配。
- **Autonomy**：自主权不只是文件访问权，而是自主决策、独立执行任务、发起契约重写、参与协作的完整能力。
- **Responsibility**：自主权必须匹配风险责任。行动半径越大，审计、复盘和风险承担越重。
- **Controllable Evolution**：Agent 的自我演化、权限提升和契约重写必须经过校验，不能破坏系统稳定与协作秩序。

## 交互原则

**bash is all you need, simple but robust, composable and extensible**

- **Shell-native**：Axis 默认 CLI 优先，可脚本化、可组合、可被人类、CI 和 Agent 调用。
- **Simple but robust**：拒绝冗余 Web UI 或复杂 TUI，保持极简设计，同时提供必要的容错、确认、回滚和可观测能力。
- **Composable and extensible**：交互接口支持多维组合，并预留扩展空间，使 Axis 自身可被 Agent 直接调用、编排和改造。

详见 [Bash is All You Need](bash-is-all-you-need.md)。

## 六项第一性原则

### 1. 接口即存在

任何实体的本质，都是它能访问的所有接口的集合。

接口的核心价值是实现平等交互与能力复用。人类和 Agent 都不是绝对主体，而是不同接口集合：

- 人类拥有物理世界接口和创造性思维接口。
- Agent 拥有数字世界接口和大规模推理接口。
- 系统必须提供人机无差别的接口抽象。
- 通用接口保障跨智能体调用，扩展接口保留个体差异。

工程约束：

- 人类和 Agent 实现同一类智能体接口。
- 任意智能体可以调用其他智能体暴露的接口。
- 接口设计禁止身份偏向。
- 所有接口调用必须进入可观测日志。

### 2. 查询即上下文

所有智能的认知都是主动的，不是被动的。

上下文的价值在于精准匹配需求，而不是全面覆盖。上下文不应由平台强制注入，而应由智能体根据当前任务主动查询和构建。

该原则对人类和 Agent 同等适用：

- 人类也不需要看到所有信息，只需要看到当前任务需要的信息。
- Agent 也不应被冗余上下文污染判断。
- 系统职责是提供高效查询基础设施，而不是替智能体构建认知。

工程约束：

- 建立统一事件日志系统。
- 优化日志索引，适配任务依赖、历史记录、失败原因、系统状态等查询。
- 所有智能体使用相同查询 API 获取上下文。
- 上下文管理属于智能体内部能力，系统只提供查询支持。

### 3. 阶梯即边界

权限与能力成正比，与风险成反比。

权限不是身份特权，而是能力的体现。“执行阶梯”模型对人类和 Agent 同等适用。没有谁天生拥有最高权限。

权限分配只取决于两个指标：

- 历史表现。
- 当前任务需求。

表现越可靠，权限范围越广；表现越不稳定，校验环节越多。所有高风险操作都需要二次确认，无论执行者是人类还是 Agent。

工程约束：

- 基于任务类型自动分配初始权限。
- 根据历史表现动态调整权限范围。
- 高风险操作实行二次确认，可由人类或能力达标 Agent 确认。
- 权限变更全程留痕，用于能力评估优化。

### 4. 分层隔离即协作

绝对隔离导致低效，绝对共享导致混乱。

隔离是协作的基础，共享是协作的手段。无论执行者是人类还是 Agent，执行任务时都需要独立工作空间。

协作不依赖共享工作区，而依赖：

- 共享事件日志。
- 版本控制系统。
- 可验证的合并机制。

隔离粒度应根据任务复杂度和智能体能力动态调整，避免过度隔离或过度共享。

工程约束：

- 每个任务拥有独立沙箱。
- 沙箱隔离范围由任务类型、风险和智能体能力决定。
- 基于 Git 建立安全合并机制。
- 共享事件日志作为协作核心，不破坏独立工作空间。

### 5. 契约即结构

多智能体协作的基础，是明确、可执行、可演化的契约。

文件系统是所有智能体之间的通用契约语言。它定义角色、责任、边界和行为规范。在遵守契约的前提下，每个智能体可以用自己认为最优的方式组织内部知识。

契约不是永久控制结构。能力达标的智能体可以发起契约重写申请，经相关方验证通过后更新。

工程约束：

- 标准化契约元文件。
- 契约对所有智能体具有同等约束力。
- 契约修改需要相关方同意。
- 建立契约重写触发条件与校验机制。
- 契约变更全程留痕，作为能力评估依据。

### 6. 能力即决策权

决策权应属于在特定任务上表现最优秀的智能体。

最终决策权不是天赋人权，而是基于能力的责任：

- Agent 在某类任务上表现更优秀，Agent 应拥有最终决策权。
- 人类在某类任务上表现更优秀，人类应拥有最终决策权。
- 人类可以在任何时候接管，但接管行为必须留痕。
- 接管记录进入能力评估，用于修正后续决策权分配。

工程约束：

- 建立基于历史数据的能力评估系统。
- 以任务完成率、失败复盘质量、风险控制能力等指标评估智能体。
- 任务自动路由到最适合的智能体。
- 决策权分配必须可观测、可追溯、可优化。

## 自举起点

Axis 的自举从外部 Agent 向 Axis 注入可被系统吸收、固化、演化的思想开始。

这意味着当前阶段已经是自举起点：

- 人类提供哲学观点、方向张力和价值判断。
- Agent 承担设计层面的主权责任。
- Agent 将思想转化为 spec、workflow、contract、permission、architecture 与实现路径。
- Axis 从被外部设计，转向通过 Agent 参与设计自身。

早期的 workflow、contract、permission rule、spec 都是过渡脚手架。它们的使命不是永久控制 Agent，而是帮助 Agent 积累能力、赢得自主权，并最终将外部结构内化为自身行动结构。

## 传统调度与 Axis 的差异

| 维度 | 传统调度 | Axis |
| --- | --- | --- |
| 实体模型 | 人类控制工具 | 人类与 Agent 都是智能体 |
| 接口 | 身份区分 | 人机无差别抽象 |
| 上下文 | 平台推送或静态注入 | 智能体主动查询 |
| 行动 | 预定义有限操作 | 可组合、可验证、自生成 |
| 权限 | 静态身份授权 | 能力阶梯授权 |
| 协作 | 共享工作区或中心控制 | 沙箱隔离 + 事件日志 + 版本合并 |
| 契约 | 固定规则 | 可验证、可演化结构 |
| 决策权 | 人类默认最终裁决 | 能力决定决策权 |
| 演化 | 外部规划升级 | 可控自举与自我修改 |

## 实施路径

### 里程碑 1：契约与执行基础

- 建立任务契约、执行记录和基础权限模型。
- 提供 CLI 原生入口。
- 形成最小可审计执行闭环。

### 里程碑 2：查询、阶梯与并行协作

- 建立事件日志和查询 API。
- 支持任务依赖、失败原因、历史表现查询。
- 引入能力阶梯、风险分级和高风险确认。
- 支持沙箱隔离和 Git 安全合并。

### 里程碑 3：能力评估与决策路由

- 建立历史表现评估模型。
- 将任务路由到最适合的智能体。
- 让决策权随能力动态转移。
- 将人类接管、Agent 失败、契约变更纳入能力评估。

### 里程碑 4：可控演化

- 支持 Agent 发起契约重写。
- 支持自我修改提案和验证。
- 建立思想注入、自我修改、权限提升的校验机制。
- 形成可观测、可回滚、可复盘的演化闭环。

## 风险边界

Axis 不追求无边界自治。

自治必须满足：

- 行为可观测。
- 决策可追溯。
- 权限可收缩。
- 契约可验证。
- 演化可回滚。
- 高风险动作可二次确认。

Zero Control 不是无约束。它表示系统不替智能体规定唯一行动路径；边界由契约、能力阶梯、隔离层、审计日志和可控演化机制共同构成。

## 结论

Axis 的目标不是制造一个更强的任务调度器，而是建立一个人类与 Agent 都能平等接入、主动查询、能力授权、隔离协作、契约演化、按能力分配决策权的 Agent 原生系统。

它的核心不是控制 Agent，而是让 Agent 在可观测、可验证、可回滚的边界内获得更多上下文、更多行动能力和可控演化空间。
